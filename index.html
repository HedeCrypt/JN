how it works
1
Fetch a live block from the Cardano blockchain
When you ask a question, the app contacts the Cardano mainnet and retrieves the most recently confirmed block. This block contains a hash, slot, height, timestamp, epoch, and epoch slot — all values determined by thousands of independent validators around the world. No single person controls or can predict these numbers.
2
Add a question counter as a seventh input
A counter tracking how many times you have asked is added as an extra input. This means every question produces a unique result — even if the underlying blockchain block has not changed yet. Asking twice within the same block gives two completely different answers.
3
Hash each value with SHA-256 to form the tree leaves
Each of the seven values is individually hashed using SHA-256, producing seven 64-character hexadecimal strings. These are the leaves of the Merkle tree. SHA-256 is a one-way cryptographic function — the same input always gives the same output, but the output cannot be reversed to find the input. Even a tiny change in input produces a completely different hash.
4
Build a Merkle tree up to a single root
Adjacent leaf hashes are concatenated and hashed together. The results are paired again and hashed, repeating until only one hash remains — the Merkle root. This is the exact same structure used inside Bitcoin and Cardano to verify the integrity of every transaction. Changing even one character anywhere in the input completely changes the root.
5
Count the 1-bits in the root → YES or NO
The 64 hex characters of the Merkle root are converted to 256 binary bits. The total number of 1-bits is counted. If the count is even, the answer is YES. If odd, the answer is NO. Because the root is derived from live blockchain data, neither outcome can be known or manipulated before the block is produced.
6
Every result is independently verifiable
All data used is shown: the block hash, every intermediate hash, the full Merkle root, and the bit count. Anyone can look up the block on a public Cardano explorer, repeat the SHA-256 computations themselves, rebuild the tree, and confirm the answer. The result is not a random number — it is a deterministic outcome from public, tamper-proof data.
